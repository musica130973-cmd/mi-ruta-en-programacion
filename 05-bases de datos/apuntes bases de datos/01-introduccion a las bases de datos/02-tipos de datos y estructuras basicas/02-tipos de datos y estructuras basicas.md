**2\. TIPOS DE DATOS Y ESTRUCTURAS BÁSICAS**

**🧩 1. ¿Qué son los tipos de datos?**

Los **tipos de datos** son categorías que definen **qué clase de información** puede almacenar una variable o un campo (números, texto, fechas…). Sirven para que el sistema sepa **cómo interpretar, almacenar y operar** con esos valores.

**🔢 2. Tipos de datos más comunes**

**A) Tipos numéricos**

-   **Enteros (int)** Números sin decimales. Ej.: 3, –15, 2024.
-   **Reales o de coma flotante (float, double)** Números con decimales. Ej.: 3.14, –0.5.

**B) Tipo texto**

-   **Cadenas de caracteres (string, char)** Representan palabras, frases o caracteres individuales. Ej.: "Hola", "A", "1234" (aunque parezca número, es texto).

**C) Tipo lógico**

-   **Booleano (bool)** Solo puede tomar dos valores: **true** o **false**. Muy usado en condiciones y decisiones.

**D) Tipo fecha y hora**

-   **Date**, **Time**, **DateTime** Permiten trabajar con calendarios, horarios y marcas de tiempo.

**E) Tipos especiales**

-   **Null**: ausencia de valor.
-   **Binary**: datos en formato binario (imágenes, archivos).
-   **Enumerados (enum)**: conjunto de valores predefinidos.

**🧱 3. ¿Qué son las estructuras de datos?**

Son **formas de organizar y almacenar datos** para que puedan usarse de manera eficiente. Cada estructura tiene ventajas según el tipo de problema que se quiera resolver.

**📦 4. Estructuras de datos básicas**

**A) Arrays o vectores**

-   Conjunto de elementos **del mismo tipo**.
-   Acceso rápido por índice.
-   Tamaño fijo en muchos lenguajes.
-   Ej.: \[10, 20, 30\].

**B) Listas**

-   Colecciones ordenadas de elementos.
-   Suelen tener **tamaño dinámico**.
-   Permiten insertar o eliminar elementos fácilmente.
-   Ej.: lista de tareas.

**C) Pilas (Stack)**

-   Estructura **LIFO** (Last In, First Out).
-   El último en entrar es el primero en salir.
-   Ejemplo típico: deshacer acciones en un editor.

**D) Colas (Queue)**

-   Estructura **FIFO** (First In, First Out).
-   El primero en entrar es el primero en salir.
-   Ejemplo: cola de impresión.

**E) Diccionarios o mapas (Map / HashMap)**

-   Almacenan pares **clave–valor**.
-   Permiten búsquedas muy rápidas.
-   Ej.: {"nombre": "Ana", "edad": 20}.

**F) Conjuntos (Set)**

-   Colecciones **sin elementos repetidos**.
-   Útiles para comprobar pertenencia o eliminar duplicados.

**🧬 5. Estructuras más avanzadas (solo para tenerlas ubicadas)**

Aunque no son básicas, conviene conocerlas:

-   **Árboles** (como los binarios o AVL).
-   **Grafos** (nodos conectados por aristas).
-   **Tablas hash** (base de muchos diccionarios).
-   **Heaps** (para prioridades).

**🛠️ 6. ¿Por qué son importantes?**

-   Permiten **optimizar el rendimiento** de programas y bases de datos.
-   Ayudan a **organizar la información** de forma lógica.
-   Facilitan operaciones como **búsquedas, ordenaciones, inserciones** y **eliminaciones**.
-   Son la base de algoritmos y estructuras más complejas.

**📝 7. Resumen final**

-   Los **tipos de datos** definen **qué clase de información** se puede almacenar.
-   Las **estructuras de datos** definen **cómo se organiza** esa información.
-   Elegir bien ambos es clave para crear sistemas eficientes, ordenados y fáciles de mantener.
