1\. QUÉ ES UNA BASE DE DATOS

**🧩 1. Definición básica**

Una **base de datos** es un **conjunto organizado de información** que se almacena de forma estructurada para que pueda **consultarse, gestionarse y actualizarse** de manera eficiente.

En otras palabras, es un sistema que permite **guardar datos** y **acceder a ellos rápidamente** cuando se necesitan.

**🗂️ 2. ¿Para qué sirve una base de datos?**

Las bases de datos se utilizan para:

-   **Almacenar información** de forma segura y ordenada.
-   **Evitar duplicidades** y errores.
-   **Facilitar búsquedas rápidas** (por ejemplo, encontrar un cliente por su DNI).
-   **Permitir actualizaciones** sin perder coherencia.
-   **Compartir datos** entre varios usuarios o aplicaciones.
-   **Analizar información** para tomar decisiones.

**🧱 3. Elementos principales de una base de datos**

| **Elemento** | **Descripción** |
| --- | --- |
| **Datos** | La información en sí (nombres, fechas, números…). |
| **Tablas** | Estructuras donde se organizan los datos en filas y columnas. |
| **Registros** | Cada fila de una tabla; representa una unidad de información. |
| **Campos** | Cada columna; define un tipo de dato (nombre, edad, precio…). |
| **Claves** | Atributos que identifican registros o relacionan tablas. |
| **Consultas** | Peticiones para obtener información concreta. |
| **Índices** | Estructuras que aceleran las búsquedas. |

**🧠 4. Tipos de bases de datos**

**a) Según su estructura**

-   **Relacionales (SQL)** Organizan la información en tablas relacionadas entre sí. Ejemplos: MySQL, PostgreSQL, Oracle, SQL Server.
-   **No relacionales (NoSQL)** Diseñadas para grandes volúmenes de datos y estructuras flexibles. Tipos: documentos, grafos, clave-valor, columnas. Ejemplos: MongoDB, Redis, Cassandra.

**b) Según su ubicación**

-   **Locales**: instaladas en un ordenador.
-   **En red**: accesibles desde varios equipos.
-   **En la nube**: gestionadas por servicios como AWS, Azure o Google Cloud.

**🔐 5. Características importantes**

Una buena base de datos debe cumplir:

-   **Integridad**: los datos deben ser correctos y coherentes.
-   **Seguridad**: control de accesos y protección contra pérdidas.
-   **Disponibilidad**: accesible cuando se necesite.
-   **Escalabilidad**: capacidad de crecer sin perder rendimiento.
-   **Redundancia mínima**: evitar duplicar información innecesariamente.

**🛠️ 6. Sistemas Gestores de Bases de Datos (SGBD)**

Un **SGBD** es el software que permite crear, administrar y consultar bases de datos.

Ejemplos conocidos:

-   MySQL
-   PostgreSQL
-   Oracle Database
-   Microsoft SQL Server
-   SQLite
-   MongoDB (NoSQL)

**📌 7. Ejemplos de uso en la vida real**

-   Bancos: cuentas, movimientos, clientes.
-   Hospitales: historiales médicos.
-   Redes sociales: perfiles, publicaciones, mensajes.
-   Tiendas online: productos, pedidos, usuarios.
-   Colegios: notas, matrículas, horarios.

**📝 8. Resumen final**

Una base de datos es un sistema organizado para **almacenar, gestionar y recuperar información** de forma eficiente. Es esencial en prácticamente cualquier aplicación moderna, desde una web sencilla hasta sistemas empresariales complejos.
