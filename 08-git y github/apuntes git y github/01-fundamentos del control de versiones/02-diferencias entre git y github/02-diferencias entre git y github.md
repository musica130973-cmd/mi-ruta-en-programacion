**2\. DIFERENCIAS ENTRE GIT Y GITHUB**

Aunque muchas personas los confunden, **Git y GitHub no son lo mismo**. Git es una herramienta de control de versiones, mientras que GitHub es una plataforma online que usa Git para almacenar y gestionar proyectos.

Vamos a ver sus diferencias de forma clara y ordenada.

**📌 1. ¿Qué es Git?**

**Git es un sistema de control de versiones distribuido (DVCS).**

**✔ Características principales**

-   Se instala en tu ordenador
-   Funciona sin conexión a internet
-   Permite crear repositorios locales
-   Gestiona el historial de cambios
-   Permite crear ramas, fusionarlas, revertir cambios, etc.
-   Es rápido, seguro y muy eficiente

**✔ Qué puedes hacer con Git**

-   Crear un repositorio (git init)
-   Guardar cambios con commits
-   Crear y gestionar ramas
-   Ver el historial del proyecto
-   Fusionar cambios
-   Recuperar versiones anteriores

👉 **Git = herramienta local para gestionar versiones.**

**📌 2. ¿Qué es GitHub?**

**GitHub es una plataforma web que aloja repositorios Git en la nube.**

**✔ Características principales**

-   Necesita internet
-   Permite almacenar repositorios remotos
-   Facilita la colaboración entre desarrolladores
-   Añade herramientas extra que Git no tiene

**✔ Qué ofrece GitHub**

-   Repositorios remotos
-   Pull Requests
-   Issues (gestión de tareas)
-   Wikis
-   GitHub Actions (automatización CI/CD)
-   Gestión de equipos y permisos
-   Visualización del historial y ramas
-   Portafolios con GitHub Pages

👉 **GitHub = servicio online que usa Git para trabajar en equipo.**

**📌 3. Diferencias clave entre Git y GitHub**

| **Característica** | **Git** | **GitHub** |
| --- | --- | --- |
| Tipo | Herramienta de control de versiones | Plataforma web |
| Instalación | Se instala en tu PC | No se instala, se usa online |
| Conexión | Funciona sin internet | Necesita internet |
| Repositorios | Locales | Remotos |
| Colaboración | Limitada | Muy completa |
| Funciones extra | No | Sí (PRs, Issues, Actions, Wikis…) |
| Propósito | Gestionar versiones | Compartir y colaborar |

**📌 4. ¿Cómo trabajan juntos?**

Git y GitHub se complementan:

1.  **Git** gestiona los cambios en tu ordenador.
2.  **GitHub** almacena esos cambios en la nube.
3.  Usas comandos como:
    -   git push → subir cambios a GitHub
    -   git pull → descargar cambios de GitHub
    -   git clone → copiar un repositorio remoto

👉 Git = motor 👉 GitHub = garaje donde guardas el coche y lo compartes con otros

**📌 5. ¿Puedo usar Git sin GitHub?**

Sí, perfectamente. Puedes trabajar solo con repositorios locales.

**📌 6. ¿Puedo usar GitHub sin Git?**

No. GitHub está basado en Git, así que necesitas Git para interactuar con él.

**🎯 Resumen final**

-   **Git** es la herramienta que gestiona versiones.
-   **GitHub** es la plataforma donde guardas y compartes tus repositorios Git.
-   Git funciona localmente; GitHub funciona en la nube.
-   Git es el motor; GitHub es el ecosistema colaborativo.
