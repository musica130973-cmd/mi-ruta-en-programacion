**TEMARIO GIT Y GITHUB**

**1\. Fundamentos del control de versiones**

-   Qué es el control de versiones y por qué es esencial.
-   Diferencias entre Git y GitHub.
-   Repositorios locales vs. remotos.
-   Conceptos clave: commit, branch, merge, snapshot, staging area.

**2\. Instalación y configuración inicial**

-   Instalación de Git en Linux, Windows y macOS.
-   Configuración básica (git config).
-   Generación y uso de claves SSH para GitHub.
-   Autenticación con HTTPS y tokens personales.

**3\. Flujo básico de trabajo con Git**

-   Crear un repositorio (git init).
-   Añadir archivos al staging (git add).
-   Crear commits (git commit).
-   Ver historial (git log, git show).
-   Ignorar archivos con .gitignore.

**4\. Repositorios remotos y GitHub**

-   Crear repositorios en GitHub.
-   Conectar repositorio local con remoto (git remote).
-   Subir cambios (git push).
-   Descargar cambios (git pull, git fetch).
-   Clonar repositorios (git clone).

**5\. Ramas y flujos de trabajo**

-   Crear y cambiar de rama (git branch, git checkout, git switch).
-   Fusionar ramas (git merge).
-   Resolver conflictos.
-   Estrategias de branching: Git Flow, GitHub Flow, Trunk Based Development.
-   Buenas prácticas para nombrar ramas.

**6\. Pull Requests y colaboración**

-   Qué es un Pull Request y cuándo usarlo.
-   Crear, revisar y aprobar PRs.
-   Comentarios, sugerencias y discusiones.
-   Revisiones de código efectivas.
-   Merge vs. Squash vs. Rebase en GitHub.

**7\. Reescritura de historial y operaciones avanzadas**

-   git reset (soft, mixed, hard).
-   git revert vs. git reset.
-   git rebase y rebase interactivo.
-   Limpieza de ramas y commits.
-   Recuperación de cambios con git reflog.

**8\. Gestión de proyectos en GitHub**

-   Issues: creación, etiquetas, asignación y cierre.
-   Milestones y proyectos.
-   Tableros Kanban con GitHub Projects.
-   Templates de issues y PRs.
-   Wiki del repositorio.

**9\. GitHub Actions (automatización)**

-   Introducción a CI/CD.
-   Workflows básicos.
-   Automatizar tests, builds y despliegues.
-   Uso de acciones predefinidas.
-   Variables de entorno y secretos.

**10\. Seguridad en GitHub**

-   Gestión de claves y tokens.
-   Dependabot y alertas de seguridad.
-   Protección de ramas.
-   Revisiones obligatorias.
-   Firmar commits (GPG).

**11\. Integración con otras herramientas**

-   GitHub Desktop.
-   Extensiones de GitHub para VS Code.
-   Integración con Docker, plataformas cloud y servicios externos.
-   Webhooks.

**12\. Buenas prácticas profesionales**

-   Mensajes de commit claros y consistentes.
-   Convenciones de estilo (Conventional Commits).
-   Documentación del repositorio (README, LICENSE).
-   Versionado semántico (SemVer).
-   Mantener un historial limpio y entendible.

**13\. Proyectos prácticos**

-   Crear un repositorio completo desde cero.
-   Colaborar en un proyecto real mediante PRs.
-   Automatizar tests con GitHub Actions.
-   Crear un portafolio personal alojado en GitHub Pages.
-   Gestionar un proyecto con Issues + Projects.

**14\. Errores comunes y cómo evitarlos**

-   Hacer commits demasiado grandes.
-   Trabajar en main en lugar de ramas.
-   No documentar cambios.
-   Subir archivos sensibles.
-   No resolver conflictos correctamente.
